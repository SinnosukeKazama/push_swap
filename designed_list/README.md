## 設計
### 検証:基数ソート(radix sort,以下rs)。
- 条件点:任意のI, n=256, INT_MIN<= In <=INT_MAX.
- 比較点:2ケースでのソート速度を比べる。
 - 1)基数=10, 桁数=10(Inの値範囲から。).
  - ビット演算でなく除算を使うため遅い。
  - rsの設計上、バケツ[基数][n];の配列を用意する必要ある。そのため、1)バケツ[10][256]でバケツn=2560.これは2)と比べると少ない。よってメモリアクセスの時間が少ない(速い)。
 - 2)基数=256, 桁数=4(同上,すなわち桁数=byteで,int型=4byteから.).
  - 除算の代わりにビット演算を使う為、速い。
  - 2)バケツ[256][256]=65536.これは1)と比べると多い。よってメモリアクセスの時間が多い(遅い).
- 結果
 - 1) 18873 [ns]
 - 2) 61614 [ns]
- 結果からわかること
 - ソート完了まで、2)は1)の約3倍の時間がかかった。
 - ビット演算より、メモリアクセスの時間の方がはるかに影響している。
